pipHelp
service in module pipHelp.Service

Description

This service is provided an interface to manage the Help component. It is available on the config and run application phases. On the both phases the interface is the same. This module requires the 'pipState' module.

Dependencies

  • pipState

Methods

  • addPage(pageObj)

    This method allows add new page into navigation menu. It accepts config object to define new state with needed params.

    Parameters
    ParamTypeDetails
    pageObjObject

    Configuration object contains settings for another page

    pageObj.stateObject.<string>

    Name of page state which is available via UI router

    pageObj.titleObject.<string>

    Page title in the navigation menu.

    pageObj.accessObject.<boolean>

    If it is true it will be available only for logged in users

    pageObj.visibleObject.<boolean>

    If it is true the page will be visible

    pageObj.stateConfigObject.<Object>

    Configuration object in format like UI Router state

    Example

    pipHelpProvider.addPage({
    state: 'test',
    title: 'Test help page',
    auth: true,
    stateConfig: {
        url: '/test',
        templateUrl: 'help/help_test1.html'
    }
});

  • getDefaultPage()

    This method return name of the default state.

    Returns
    string

    Name of the state

    Example

    pipHelpProvider.getDefaultPage();

  • getPages()

    This method returns asset of all pages registered in the Help component.

    Returns
    Array<Object>

    List of registered states

    Example

    pipHelpProvider.getPages();

  • setDefaultPage(name)

    This method establishes passed state as default which is redirected at after transfer on abstract state

    Parameters
    ParamTypeDetails
    nameObject

    Name of the state

    Example

    pipHelpProvider.setDefaultPage('test');
